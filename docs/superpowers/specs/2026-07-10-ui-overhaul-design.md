# UI 전면 개선 설계안 — 편집실·결과 화면 (shadcn + DESIGN.md 방법론)

작성: 2026-07-10 · 상태: 검토 대기
근거 문서: [getdesign.md](https://getdesign.md/) · [shadcn.io/design](https://www.shadcn.io/design) · `docs/DESIGN.md`(이 설계와 함께 신설되는 디자인 시스템 단일 출처)

## 0. 문제 정의 — 실측 감사 (2026-07-10, WebKit 390×844)

| # | 문제 | 증거 |
|---|------|------|
| P1 | 결과 진입 시 편집기가 화면 밖 — 마스트헤드(로고+lede)와 메타 문단이 첫 화면을 차지 | 진입 스크린샷에서 플레이어 미노출 |
| P2 | "완성" 제목이 세로로 깨짐(완/성), 메타·zip 버튼 겹침 | 헤더가 flex 한 줄에 과적 |
| P3 | **도구 탭을 눌러도 패널이 안 보임** — sticky 플레이어와 도구바 사이 패널로 스크롤 연결 없음 | 음악/썸네일 탭 스크린샷이 동일 |
| P4 | 컴포넌트 파편화 — chips·style-picks·pos-off·toggle이 제각각의 padding/폰트/보더 | style.css에 유사 버튼 스타일 4벌 |
| P5 | 적용 버튼이 무엇을 적용하는지 안 보임 — dirty 상태·변경 요약 없음, 미변경에도 항상 활성 | |
| P6 | 적용 시 편집실이 통째로 진행 화면으로 교체 — 편집 컨텍스트 소실 | showError/progress 전환 로직 |
| P7 | 산출물(다운로드·공유·비교)이 편집실 아래에 파묻힘 | 풀페이지 스크린샷 |
| P8 | 세로 영상 좌우 레터박스가 커서 플레이어 영역 낭비 | 38vh 고정 |

## 1. 설계 원칙 (이 순서로 충돌을 판정)

1. **결과가 첫 화면** — 진입 즉시 플레이어가 보이고 재생 가능해야 한다.
2. **모든 조작은 보이는 곳에서 일어난다** — 탭을 누르면 그 패널이 반드시 뷰포트 안에 나타난다(시트).
3. **변경은 추적되고, 적용은 예고된다** — dirty 배지·변경 요약·미변경 시 비활성.
4. **컴포넌트는 한 벌** — 모든 컨트롤은 `docs/DESIGN.md`의 컴포넌트 어휘로만 조립한다.
5. **컨텍스트를 뺏지 않는다** — 재생성·업로드 등 대기는 오버레이로, 화면 전환은 사용자가 시킨 것만.

## 2. 스택 결정 — React 마이그레이션은 하지 않는다

shadcn/ui는 React+Tailwind 컴포넌트지만, 그 가치의 본체는 **토큰 체계와 컴포넌트 사용 규칙**이다(getdesign.md가 문서화하는 대상도 코드가 아니라 규칙).
- 채택: shadcn의 CSS 변수 토큰 네이밍·다크 팔레트 관례·컴포넌트 시각 규격(Button/Tabs/Sheet/Slider/Badge/Switch/Toggle Group)을 **바닐라 CSS 컴포넌트 클래스로 이식**.
- 기각: React 재작성 — 검증된 1,400줄 로직(플레이어 동기화·트림 바·큐 폴링)의 재작성 리스크가 시각 개선과 무관하게 크고, 빌드 체인이 없는 현재 배포(정적 서빙)의 단순성을 해친다.
- 브랜드: Reel Room의 다크 시네마 아이덴티티(앰버 액센트·필름 모티프·Fraunces 마스트헤드)는 유지하되, **면·보더·간격 체계를 shadcn 문법으로 재구축**한다. 값은 `docs/DESIGN.md`가 단일 출처.

## 3. 정보 구조 재설계

### 3.1 결과 화면 = 편집실이 곧 화면

```
[콤팩트 앱바]  ← 새로 | ● 잡 이름 · 상태     ⓘ 메타 | ↓ 내보내기
[플레이어]     세로 영상이면 blur 배경 채움, 높이 42vh
[트랜스포트]   ⏵ 0:03.7/0:12.0        [적용 · 3]  ← dirty 배지
[타임라인]     필름스트립 + 플레이헤드 (유지)
[도구바]       클립 | 자막 | 음악 | 썸네일 | 효과 | 내보내기
[도구 시트]    도구바 위로 올라오는 bottom sheet (max 48vh, 내부 스크롤)
```

- **마스트헤드 제거(결과 화면에서)**: 로고는 앱바 좌측 점 아이콘으로. 업로드 화면만 풀 마스트헤드 유지.
- **메타 정보**: "6개 소스 결합 · 구간 6개 · LLM $0.003 · 노트…" 문단 → 앱바 ⓘ 아이콘 → 시트에 표 형태.
- **산출물 그리드 → "내보내기" 탭**: 다운로드 카드·공유·클린 비교·zip·srt 전부 내보내기 시트로 이동(P7 해소). 결과 페이지 하단의 세로 나열 삭제.
- **최근 작업 패널**: 데스크톱 우측 고정 유지, 모바일은 앱바 잡 이름 탭 → 시트.

### 3.2 도구 패널 = Bottom Sheet (P3 해소)

- 도구바 탭 → 패널이 도구바 위로 **시트로 슬라이드 업** (스크롤 위치와 무관하게 항상 보임).
- 같은 탭 재탭 or 시트 그립 아래로 드래그 → 닫힘. 시트가 열려도 플레이어·타임라인은 위에 남는다(좁으면 플레이어가 축소 높이 28vh로).
- 클립 탭은 예외적으로 타임라인 셀 탭과 연동: 셀 선택 시 자동으로 클립 시트 오픈.
- 데스크톱(≥900px): 시트 대신 **우측 사이드 패널 360px 고정** — 좌측(플레이어+타임라인) / 우측(도구 패널+도구바 세로형).

### 3.3 적용(재생성) 흐름 (P5·P6 해소)

- **dirty 추적**: 편집 상태(구간·자막·음악·썸네일·효과)가 마지막 렌더와 다르면 변경 카운트 배지 — `적용 · 3`. 미변경이면 비활성.
- 적용 버튼 롱프레스/우측 ⌄ → 변경 요약 리스트("구간 2 트림 1.0→2.0s", "BGM: 자동→calm/…", "자막: 페이드→바운스").
- 재생성 중: 화면 전환 없이 **플레이어 위 진행 오버레이**(프로그레스 링+단계 텍스트). 타임라인·패널은 dim+잠금. 완료 시 플레이어 소스만 교체 + 토스트 "적용 완료".
- 실패 시 오버레이에 에러+재시도, 편집 상태는 그대로 보존.

### 3.4 업로드·진행 화면

- 업로드: 현 구조 유지(드롭존→세부 설정→CTA), 토큰·컴포넌트만 신체계로 교체.
- 진행: 결과 화면과 같은 셸(앱바+게이트 자리)에 프로그레스 — 완료 시 같은 자리에서 플레이어로 전환(화면 점프 제거).

## 4. 인터랙션 규칙

- 터치 타겟 최소 44×44px (현재 도구바·칩 일부 미달).
- 포커스: 모든 인터랙티브 요소에 `--ring` 2px 링 (`:focus-visible`).
- 시트: 열림 240ms cubic-bezier(0.32,0.72,0,1), 배경 dim 없음(플레이어 확인이 목적이므로).
- 토스트(우하단/모바일 상단): 적용 완료·B컷 업로드·비밀번호류 상태 텍스트를 `edit-status` 한 줄에서 토스트로 승격.
- 가로 스크롤 줄(썸네일 옵션 등)의 끝 페이드 유지 — 단 토큰 색으로.
- `prefers-reduced-motion` 존중(기존 규칙 유지).

## 5. 마이그레이션 계획 — 각 단계가 독립 배포 가능

| 단계 | 내용 | 검증 |
|---|---|---|
| M1 | `docs/DESIGN.md` 신설 + tokens.css(shadcn 변수) 도입, 기존 변수 매핑 치환 | 스크린샷 회귀(업로드·진행·결과) |
| M2 | 컴포넌트 클래스 한 벌(btn/seg/switch/field/card/badge/sheet/toast)로 기존 4벌 통합 | 컨트롤 전수 교체 후 WebKit 인터랙션 스위트 재실행 |
| M3 | 결과 화면 IA 재배치 — 앱바·마스트헤드 제거·내보내기 탭·메타 시트 | P1·P2·P7 스크린샷 대조 |
| M4 | 도구 시트 + 데스크톱 사이드 패널 | P3 재현 테스트(탭→패널 가시) |
| M5 | dirty-state·적용 오버레이·토스트 | P5·P6 시나리오 테스트 |

## 6. 명시적 비범위

- React/프레임워크 전환, 서버 렌더 파이프라인 변경, 새 편집 기능 추가(이 설계는 순수 UI/UX).
- PIL 자막 엔진 UI 노출 확대.

## 7. 리스크

- 시트 제스처(드래그 닫기)는 iOS Safari 주소창 제스처와 간섭 가능 → 그립 탭+탭 재탭 닫기를 기본, 드래그는 보조.
- 산출물을 내보내기 탭으로 옮기면 "결과가 어디 갔지" 순간이 올 수 있음 → 최초 완료 시 내보내기 탭에 점 배지 + 완료 토스트에 "내보내기에서 다운로드" 문구.

# 몽타주 숏츠 콘텐츠 인지 트림 — 설계

날짜: 2026-07-13
상태: 승인됨

## 문제

몽타주 경로(총 길이 ≤ 목표)의 숏츠가 내용을 보지 않고 잘린다:

1. `pipeline.py` 몽타주 분기가 **무조건 클립당 2초로 트림**한다(`montage_seconds=2.0` 고정).
   10초 영상 1개를 올려도 씬 경계로 쪼갠 뒤 각각 2초만 남긴다. 총량이 이미
   숏츠 길이에 맞아도 자른다 — "전체 내용이 다 필요한 영상"을 구분하지 못한다.
2. `auto_cut.py:trim_montage_segments`가 어느 창을 남길지를 **ffmpeg 프레임
   변화량(모션)만으로** 고른다. 카메라 흔들림·전환 순간이 점수가 높아 엉뚱한
   조각이 뽑힌다. 내용(무엇이 찍혔는지, 무슨 말인지)은 보지 않는다.
3. 발화가 있는 클립도 몽타주 경로에서는 단어 경계를 보지 않아 말이 중간에 잘린다.

증상(사용자 확인): 엉뚱한 짧은 조각만 나옴. 주력 입력 두 가지 모두에서 발생 —
5초 내외 클립 여러 개(몽타주), 10초 영상 1개.

## 선택한 접근

**A. 예산 기반 트림 + 기존 비전 콜 확장** (B 휴리스틱만 / C LLM 편집 디렉터 대비:
B는 무발화 클립의 내용을 여전히 못 보고, C는 현 단계에 과설계)

## 설계

### 1. 트림 판단 — 고정 2초 → 숏츠 예산 역산

`analyze()`의 몽타주 분기:

- 총량 ≤ `shorts_ideal_seconds`(기본 25초) → **트림 없음, 전체 유지**.
  (10초 영상 1개 케이스가 여기서 해결 — 10초 그대로 숏츠)
- 총량 > 25초 → 클립당 `ideal ÷ 클립 수`(최소 1.5초)를 기본 창으로 트림.
  어느 창을 남길지는 §2가 결정.
- 트림 후에도 총량 > `shorts_max_seconds`(45초) → 비례 축소로 상한 준수
  (창 최소 1.5초 유지).

`montage_seconds`는 기본값 "auto"(예산 역산)로 변경. 숫자를 명시하면 현행처럼
고정 길이 — A/B 비교용 탈출구. `0`의 "전체 유지" 의미는 유지. 웹 UI 기본값도 auto.

### 2. 어느 구간을 남길지 — 모션 → 비전 LLM (기존 콜 확장)

매 잡마다 이미 나가는 장면 자막 비전 콜(`generate_scene_captions`)을 확장:

- 클립당 프레임 1장(중앙) → **3장(초·중·후반, 25%/50%/75% 지점)** 그리드 전송.
  이미지 1장·콜 1회 유지 — 비용 증가 미미.
- 응답에 클립당 2개 필드 추가:
  - `peak`: 핵심 순간의 클립 내 상대 위치 (0.0~1.0)
  - `keep`: `whole`(내용이 이어져 통으로 필요) / `trim`(한 순간이면 충분)
- 트림 창은 `peak` 중심으로 배치, 클립 경계로 클램프.
- `keep=whole` 클립은 예산이 허용하는 한 통으로 유지 — "전체가 필요한 클립"과
  "핵심만 남길 클립"을 LLM이 클립 단위로 구분.

**폴백 계층** (기존 "키 없이 무료" 경로 불변):
- LLM 키 없음 / 콜 실패 → 현행 모션 기반 트림 그대로.
- 응답 일부만 유효 → 유효한 클립만 반영, 나머지는 모션 폴백.

### 3. 발화 클립 — 단어 경계 스냅

몽타주 speech 분기(transcript 있음)에서는 트림 창을 단어 경계에 스냅
(`snap_to_word_bounds` 재사용). 스냅으로 창이 0.8초 미만이 되면 스냅 전 유지
(기존 함수 계약).

### 4. 데이터 계약 유지

- `selection.json`: `clip_start`/`clip_end`(편집기 트림 바), `reason="montage"`
  마커(`is_montage` 판별), `captions.json` 형태 모두 불변.
- `peak`/`keep`은 segments 추가 필드로 보존 — 선정 근거 추적용.
  (`validate_segments`류 소비자는 미지 필드에 관용적)

### 5. 측정 — 판단 근거를 잡 로그에 남김

- "전체 유지(총 X초 ≤ 예산 Y초)" vs "트림(예산 Y초, 클립당 Z초)" 출력.
- 클립별 선정 근거: LLM peak / 모션 폴백 여부.
- `--montage-seconds 2`로 구버전 재현 가능 → 같은 입력 before/after A/B 비교.

### 6. 테스트

순수 함수로 분리해 `tests/test_pure.py`에 추가:

- 예산 로직: 총량 ≤ ideal → 무트림 / 초과 시 클립당 분배 / max 상한 비례 축소.
- peak 기반 창 배치 (경계 클램프 포함).
- keep=whole 존중 + 예산 초과 시 처리.
- 단어 경계 스냅.
- 불량 LLM 응답(peak 범위 밖, keep 미지 값, 클립 수 불일치) → 모션 폴백.

## 변경 파일

- `pipeline.py` — 몽타주 분기, 예산 함수(신규 순수 함수)
- `auto_cut.py` — 비전 프롬프트·응답 병합, 트림 함수(peak 지원)
- `web/app.py` — `montage_seconds` 기본값 auto
- `tests/test_pure.py` — §6 테스트
